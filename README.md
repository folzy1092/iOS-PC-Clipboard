# Clipboard

Синхронизация буфера обмена между Windows и iPhone по локальной сети.  
Текст и изображения — в обе стороны, в реальном времени через WebSocket.

## Ограничения

- **Только текст и одиночные изображения** — утилита рассчитана на быструю синхронизацию стандартного буфера обмена.
- **Один объект за раз** — если скопировать на iPhone сразу несколько фотографий, передача не сработает. В буфер ПК падает строго одна картинка.
- **Без поддержки тяжёлых файлов** — передача медиафайлов (`.mp4`, `.mp3`, `.mov`, `.zip` и т.д.) не поддерживается. Для работы с файлами нужна другая архитектура.

---

## Возможности

- **iPhone → ПК** — автоматически кладёт текст или изображение в буфер Windows.
- **ПК → iPhone** — мониторит буфер ПК и отдаёт его по запросу с шортката.
- Веб-панель с историей, ручной отправкой и переключателями.
- Значок в трее с автозагрузкой, быстрыми действиями и перезапуском.
- Работает только в локальной сети Wi‑Fi.

## Требования

- Windows 10/11.
- Python 3.9+.
- iPhone с Apple Shortcuts.

## Установка

```bash
git clone https://github.com/folzy1092/iOS-PC-Clipboard
cd iOS-PC-Clipboard
pip install -r requirements.txt
```

## Запуск

```bash
# Без окна консоли (рекомендуется)
run.bat

# Или напрямую
pythonw server.pyw
```

После запуска сервер появится в трее. Веб-панель: [http://localhost:5000](http://localhost:5000)

| Параметр | По умолчанию | Описание |
|---|---|---|
| `HOST` | `0.0.0.0` | Адрес сервера |
| `PORT` | `5000` | Порт |
| `DEFAULT_AUTO_TO_PC` | `True` | Автокопирование iPhone → ПК при старте |
| `DEFAULT_AUTO_TO_PHONE` | `True` | Автокопирование ПК → iPhone при старте |

## Apple Shortcuts

Нужны два шортката.

Перед их использованием обязательно настройте брандмауэр Windows и выберите способ подключения — по имени компьютера или по IP (см. раздел [Настройка подключения](#настройка-подключения)).

### Шорткат 1 — Отправить с iPhone на ПК

Делает POST на `/update` с контентом из буфера обмена.

- **Действие:** URL → `http://<АДРЕС_ПК>:5000/update`
- **Метод:** POST
- **Тело:** `{"data": "<base64 содержимое буфера>"}`

Готовый вариант: [To PC (iCloud)](https://www.icloud.com/shortcuts/f6e1fd9a83f848ab9469bdfe9e5295b9)

### Шорткат 2 — Получить с ПК на iPhone

Делает GET на `/get-phone` и кладёт ответ в буфер iPhone.

- **Действие:** URL → `http://<АДРЕС_ПК>:5000/get-phone`
- **Метод:** GET
- **Ответ:** `{"type": "text", "data": "..."}` или `{"type": "image", "data": "<base64>"}`

Готовый вариант: [From PC (iCloud)](https://www.icloud.com/shortcuts/8963ffa442c84585ac33677b9e8582b0)

## Настройка подключения

Чтобы iPhone мог достучаться до сервера на Windows, выполните два шага.

### Шаг 1. Откройте порт в брандмауэре Windows

Запустите командную строку **от имени администратора** и выполните команду:

```bash
netsh advfirewall firewall add rule name="Clipboard Server" dir=in action=allow protocol=TCP localport=5000
```

### Шаг 2. Выберите формат адреса для URL в Shortcuts

После импорта шорткатов замените `<АДРЕС_ПК>` в поле URL на один из вариантов.

**Вариант 1: По имени хоста**  
Рекомендуется использовать имя компьютера с суффиксом `.local`, например:  
`http://DESKTOP-NAME.local:5000`

Узнать имя своего ПК можно командой:

```bash
hostname
```

**Вариант 2: По IP-адресу**  
Используйте локальный IPv4-адрес, например:  
`http://192.168.1.45:5000`

Узнать текущий IP можно командой:

```bash
ipconfig
```

⚠️ **Примечание:** если у вас DHCP, IP-адрес компьютера может меняться после перезапуска роутера, и шорткаты придётся перенастраивать. Использование `.local` решает эту проблему.

## Автозагрузка

Правой кнопкой по иконке в трее → **Автозагрузка с Windows**.  
Записывает `pythonw server.pyw` в реестр `HKCU\...\Run`.  
Галочка рядом с пунктом показывает текущее состояние.
